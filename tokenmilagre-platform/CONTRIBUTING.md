# Contribuindo para $MILAGRE

Obrigado pelo interesse em contribuir! Este documento fornece diretrizes para contribuições.

## 🚀 Como Contribuir

### 1. Fork e Clone

```bash
git clone https://github.com/SEU_USUARIO/tokenmilagre-platform.git
cd tokenmilagre-platform
npm install
```

### 2. Crie uma Branch

```bash
git checkout -b feature/minha-feature
# ou
git checkout -b fix/correcao-bug
```

### 3. Faça suas Alterações

- Siga o padrão de código existente
- Adicione testes quando aplicável
- Atualize documentação se necessário

### 4. Commit

Usamos [Conventional Commits](https://conventionalcommits.org/):

```bash
git commit -m "feat: adiciona nova funcionalidade X"
git commit -m "fix: corrige bug na página Y"
git commit -m "docs: atualiza README"
```

### 5. Push e Pull Request

```bash
git push origin feature/minha-feature
```

Abra um Pull Request descrevendo suas alterações.

## 📋 Checklist do PR

- [ ] Código segue o estilo do projeto
- [ ] Testes passam (`npm test`)
- [ ] Build funciona (`npm run build`)
- [ ] Documentação atualizada (se aplicável)

## 🐛 Reportando Bugs

Use as [Issues](../../issues) com:

1. Descrição clara do problema
2. Passos para reproduzir
3. Comportamento esperado vs atual
4. Screenshots (se aplicável)

## 💡 Sugerindo Features

Abra uma Issue com:

1. Descrição da feature
2. Por que seria útil
3. Exemplos de uso
